XrSpatialRaycastSnapshotCreateInfoANDROID(3) Manual Page

Name

XrSpatialRaycastSnapshotCreateInfoANDROID - Information to synchronously create a snapshot handle via raycast.

C Specification

The XrSpatialRaycastSnapshotCreateInfoANDROID structure is defined as:

// Provided by XR_ANDROID_spatial_discovery_raycast
typedef struct XrSpatialRaycastSnapshotCreateInfoANDROID {
    XrStructureType                       type;
    const void*                           next;
    uint32_t                              componentTypeCount;
    const XrSpatialComponentTypeEXT*      componentTypes;
    const XrSpatialRaycastInfoANDROID*    raycastInfo;
} XrSpatialRaycastSnapshotCreateInfoANDROID;

Members

Member Descriptions

  • type is the XrStructureType of this structure.

  • next is NULL or a pointer to the next structure in a structure chain. No such structures are defined in core OpenXR or this extension.

  • componentTypeCount is a uint32_t describing the count of elements in the componentTypes array.

  • componentTypes is an array of XrSpatialComponentTypeEXT.

  • raycastInfo is the XrSpatialRaycastInfoANDROID structure describing the information of a ray.

Description

Valid Usage (Implicit)

  • The XR_ANDROID_spatial_discovery_raycast extension must be enabled prior to using XrSpatialRaycastSnapshotCreateInfoANDROID

  • type must be XR_TYPE_SPATIAL_RAYCAST_SNAPSHOT_CREATE_INFO_ANDROID

  • next must be NULL or a valid pointer to the next structure in a structure chain

  • If componentTypeCount is not 0, componentTypes must be a pointer to an array of componentTypeCount valid XrSpatialComponentTypeEXT values

  • raycastInfo must be a pointer to a valid XrSpatialRaycastInfoANDROID structure
